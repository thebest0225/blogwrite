# 블로그라이터 v2 — 확정 기획서

작성: 2026-07-17 · 상태: 확정, 단계별 구축 중

## 0. 개요
원본 초안(상세·링크 포함)을 받아 **목적지(워드프레스) + 쿠션(블로거·네이버)** 글로 재가공·발행하는 애드센스 콘텐츠 자동화 도구. MangoHub 로그인(SSO) 기반 멀티테넌시.

## 1. 멀티테넌시
- MangoHub `user_id`별로 완전 분리: 개별 목적지·개별 API 키(Anthropic·KIE)·자기 Claude 계정 MCP 연결·독립 운영.
- API 키는 서버 공용 .env → **사용자별 설정(암호화 저장)** 으로 이동. (혼자 쓰는 현재는 기존 키를 사용자 설정에 넣어둠)

## 2. 저장소: SQLite (사용자별 스코프)
| 테이블 | 내용 |
|---|---|
| users_settings | 톤/독자/저자bio/썸네일/액센트/광고코드/엔진/기본목적지/API키(암호화) |
| destinations | {name, platform(wordpress/blogger), site_url, creds(암호), is_default} — 다중 |
| drafts | {title, content, keyword, source(mcp/web/auto), status} |
| work_items | {draft_id, target(wp/blogger/naver), destination_id, title, html, status, published_url} |
| assets | {url, title, keyword, excerpt} — 연관 링크 소스 |
| schedules | {mode(preset/trend), keywords[], times_per_day, dest_id, enabled} |

## 3. 콘텐츠 로직 (기존 유지 + 강화)
- 목적지=워드프레스(가장 깊이, 광고), 쿠션=블로거·네이버(유입).
- 한 초안 → 다종 생성, 서로 차별화(다른 파생 키워드). 다중 목적지면 목적지 글도 여러 개(비슷하되 변형).
- 유형별 공식(정보/방법/비교/리뷰/리스트/이슈) + 키워드 자동 도출.
- **재가공 = ① 리디자인·정보 보강 + ② 플랫폼별 상단노출 최적화(WP=SEO/스키마, 블로거=구글노출, 네이버=네이버검색 친화) + ③ 버튼·링크(linkcard/cta) 삽입** 을 결합.
- 링크: 원본 실제 URL 우선 / 목적지=연관글 상호링크 / 블로거=링크 다수 / 네이버=공식링크 유지+목적지1+연관글1(없으면 목적지2) / 유튜브→재생 임베드(블로거·WP).
- 생성 엔진: 텍스트=Claude 공식 API(웹서치), 이미지=KIE.

## 4. 발행 방식
① 자동발행(생성 즉시) ② 검수 후 발행(버튼→자동) ③ 수동(HTML 복사)
| 대상 | 자동 | 검수후 | 복사 |
|---|:-:|:-:|:-:|
| 워드프레스 | ✅ | ✅ | ✅ |
| 블로거 | ✅(Blogger API+구글OAuth) | ✅ | ✅ |
| 네이버 | ❌ | ❌ | ✅ |

## 5. 작업 상태(작업보드 = 칸반)
초안함(new) → [선택·생성] work_item(생성됨) → [검수·편집] → (발행됨→리스트에서 제거 + assets 축적)
- 각 단계 저장 → 중단돼도 이어서. 발행 완료 시 작업리스트에서 사라짐.

## 6. 화면
작업보드(홈) / 초안함 / 목적지 관리 / 설정 / 발행 자산 / 자동화

## 7. MCP (전방위)
submit_draft, search_my_posts, list_drafts, list_work_items, request_generation(초안/쿠션 생성 지시). 웹·앱 공통 OAuth(계정별).

## 8. 자동 스케줄링
서버 크론 → (프리셋 키워드/트렌드탐색) → Claude API(웹서치) 초안 → 목적지 생성 → WP 자동발행 → assets 축적. work_item 상태로 재개.

## 9. 구축 단계
1. SQLite + 멀티테넌시 + 사용자별 설정/키 + 목적지 다중관리 + 작업보드/상태 (기존 생성·편집·발행 로직 연결)
2. 초안함↔작업보드 흐름 + 다중목적지 변형생성 + 재가공 3요소 강화
3. 블로거 자동발행(Blogger API) + 발행방식 매트릭스 완성
4. 자동 스케줄링(공식 API)
5. MCP 도구 확장 + 웹·앱 OAuth

재사용: prompts.js, html-builder.js, Claude/KIE 엔진, 편집모드, mcp.js (로직 유지, 데이터 계층만 SQLite로 교체)
