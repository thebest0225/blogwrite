# 유형별 글 작성 공식 (Writing Formula Spec)

> 블로그 자동화 크롬 확장의 **글 생성 엔진**이 사용하는 실전 사양서.
> 각 유형마다 = 트리거 → 근거 → 구조 → 형식무기 → 제목/도입부 패턴 →
> E-E-A-T 삽입 → CTA → 스키마 → 이미지 슬롯 → **LLM 프롬프트 템플릿** → 체크리스트.
> 2026 트렌드 리서치 기반. (근거 출처는 문서 하단)

작성일: 2026-07-15 · 버전: v0.2

---

## 0. 모든 유형 공통 규칙 (Global Rules)

엔진이 **어떤 유형이든 항상 강제**하는 기본기.

| # | 규칙 | 이유 |
|---|------|------|
| G1 | **결론 먼저** — 모든 H2/H3 첫 문장(50~60자)에 답부터 | AI가 이 문장을 요약에 최우선 인용 |
| G2 | **섹션 자기완결** — H2/H3 하나 = 하위질문 하나, 그 안에서 완결 | 답을 쪼개면 AI가 인용 못 함 |
| G3 | **의도-형식 매칭** — 정의=문단 / 절차=번호리스트 / 비교=표 | 형식이 의도와 맞아야 스니펫·인용 |
| G4 | **구조화 데이터(스키마)** 자동 삽입 (유형별) | 스키마 있으면 AI 인용 2~4배 |
| G5 | **E-E-A-T / 1인칭 경험** 문단 최소 1개 + 저자 bio | 경험 없는 글은 AI가 후순위 |
| G6 | **신선도**: "최종 업데이트: YYYY-MM-DD" + 최신(2026) 데이터 | 최신성 = 품질 신호 |
| G7 | **H1 1개 = 자연어 질문형**, H2 3~5개, H3 각 2~3개, 각 300~500자 | AI가 질문-답 매칭으로 회수 |
| G8 | **핵심을 서두에** — AI 회수는 약 540자에서 정체 → 중요한 건 앞에 | Top-heavy reader |
| G9 | **CTA/클릭 버튼**(인라인 스타일) 섹션 사이 배치 | 클릭·체류 유도(애드센스) |
| G10 | 본문 1,000자+, 키워드 자연 반복(도배 금지), 내부/외부 링크, 이미지 alt 키워드 | 기본 위생 |
| G11 | **사람 손질 필수** — 순수 AI 글은 최상단 노출 9%, 사람 글 80% | 발행 전 검수 전제 |

> 참고 수치: AI 오버뷰 인용의 62%는 검색 1~10위 밖 페이지에서 나옴 → **구조 최적화만 잘 해도 신생 블로그도 인용 기회 있음.**

## 1. 유형 자동 판별 로직 (엔진 동작)

키워드 입력 → LLM이 아래 순서로 유형 결정.

```
1) 트리거 키워드 매칭
   - "vs, 비교, 차이, 추천, best"  → 비교/선택형 or 리스트형
   - "best, top, 추천 N, 순위, 모음" → 리스트형
   - "방법, 하는 법, how to, 설정, 등록" → How-to형
   - "후기, 리뷰, 내돈내산, 사용기"     → 리뷰형
   - "란, 뜻, 이란, 원리, 정리"        → 정보/설명형
   - 인물/작품/실시간 이슈             → 이슈/트렌드형
2) SERP 형식 추종(권장): 대상 키워드 구글 상위 10개가 리스티클이면 리스티클,
   step-by-step이면 How-to. (구글이 선호하는 형식을 거스르면 진다)
3) 모호하면 사용자에게 유형 선택 UI 노출 (기본 추천 1개 하이라이트)
```

## 2. 유형별 공식

---

### 유형 A. 정보/설명형 (Informational / What-is)

**트리거:** "○○란", "○○ 뜻", "○○ 원리", "○○ 총정리", 개념·정의·이유 질문
**검색의도:** 빠르게 이해하고 싶다 (정보 소비)
**근거:** 정보성은 AI 요약이 가장 잘 대체하는 영역 → **AI에 인용되는 것 자체가 목표.** 통일 구조 `정의 → 사례 → 근거 → FAQ`, H2 밑 첫 50~60자 결론 먼저.

**글 구조(섹션 순서):**
1. 제목(H1, 질문형)
2. 도입부: 한 문장 정의(결론) + 이 글에서 다룰 것 3줄
3. H2 "○○란?" — 정의 문단(결론 먼저) + 핵심 요약 표
4. H2 "왜 중요/왜 그런가" — 이유·배경
5. H2 "실제 사례 / 예시" — 구체 예 (E-E-A-T)
6. H2 "자주 하는 오해 / 주의점"
7. H2 FAQ (Q&A 4~6)
8. 마무리 + 관련글 CTA

**형식 무기:** 정의 문단, 핵심 요약 표(용어·의미), 불릿, FAQ
**제목 패턴:** `○○란? 뜻·특징·예시 한눈에 총정리 (2026)`
**도입부 패턴:** "○○는 [한 문장 정의]입니다. 이 글에서는 ○○의 뜻과 [A·B·C]까지 정리했습니다."
**E-E-A-T 삽입:** "직접 정리하며 헷갈렸던 부분", 실제 예시/데이터
**CTA:** 관련 개념 글로 내부링크 버튼, 하단 심화글 버튼
**스키마:** `Article` + `FAQPage`
**이미지 슬롯:** 상단 개념 설명 이미지 1 + 사례 이미지 1

**LLM 프롬프트 템플릿:**
```
너는 한국어 SEO 정보성 블로그 작가다. 아래 규칙으로 "{키워드}" 정보/설명형 글을 써라.
- 대상 독자: {타깃}
- 반드시 결론 먼저: 각 H2/H3 첫 문장(50~60자)에 그 질문의 답을 먼저 제시.
- 구조: 정의(요약표 포함) → 이유/배경 → 실제 사례 → 오해/주의 → FAQ(Q&A 5) → 마무리.
- H1은 자연어 질문형 1개, H2 3~5개, 각 H2 아래 300~500자.
- 핵심 용어 요약 표 1개 필수, 불릿 활용, 키워드 자연 반복(도배 금지).
- 1인칭 경험/관찰 문단 1개 삽입.
- 마지막에 관련 검색어 8개, 메타 디스크립션 1개(150자 내) 출력.
- 출력은 아래 JSON 스키마로만: {제목, 메타, 섹션[], faq[], 관련검색어[], 태그[], 이미지슬롯[]}
```
**발행 전 체크:** 첫 문장 결론? 요약표 있음? FAQ 스키마? 최신 연도 표기?

### 유형 B. 방법/How-to형

**트리거:** "○○ 하는 법", "○○ 설정/등록/신청 방법", "how to"
**검색의도:** 지금 따라 해서 완료하고 싶다 (행동)
**근거:** 구글 "리스트형 피처드 스니펫"이 깔끔한 H2/번호 단계에서 추출됨. `<ol><li>` 시맨틱, 각 항목 10~20자, **5~8단계**(10+ 는 스니펫 잘 안 됨), 동사로 시작. HowTo 스키마는 화면 결과는 없지만 **절차형 AI 인용에 최고 레버리지.**

**글 구조:**
1. 제목(H1, "○○ 하는 법")
2. 도입부: 결론(몇 단계로 된다) + 준비물/전제
3. H2 "준비물 / 시작 전 확인" (있으면)
4. H2 "○○ 하는 법 (단계별)" — `<ol>` 5~8단계, 각 단계 = 동사 시작 10~20자 + 부연 1~2줄
5. H2 "자주 막히는 부분 / 오류 해결"
6. H2 "팁 / 더 빠르게 하는 법" (E-E-A-T)
7. H2 FAQ
8. 마무리 + 다음 단계 CTA

**형식 무기:** 번호 리스트(`<ol>`), 단계별 스크린샷 슬롯, 체크리스트, 경고 박스
**제목 패턴:** `○○ 하는 법 (2026 최신) — 3분 만에 따라하기`
**도입부 패턴:** "○○는 총 N단계면 끝납니다. 준비물은 [X]뿐이고, 아래 순서대로 따라 하세요."
**E-E-A-T:** "실제로 해보니 여기서 막혔다 → 이렇게 해결" 문단, 직접 스크린샷
**CTA:** 각 단계 관련 도구/링크 버튼, 하단 후속 가이드 버튼
**스키마:** `HowTo` + `FAQPage`
**이미지 슬롯:** 단계마다 스크린샷 슬롯(중요), 상단 완성 예시

**LLM 프롬프트 템플릿:**
```
너는 한국어 How-to 가이드 작가다. "{키워드}"를 따라 하면 완료되는 실전 가이드를 써라.
- 대상: {타깃}, 난이도: {초보/중급}
- 핵심 절차는 5~8단계 번호 리스트(<ol>). 각 단계는 동사로 시작, 제목 10~20자 + 부연 1~2문장.
- 도입부에 "총 N단계", 준비물 명시.
- "자주 막히는 부분/오류 해결" 섹션 필수(각 문제=소제목+해결 1~2줄).
- 단계마다 [스크린샷 슬롯: 무엇을 찍을지] 표기.
- 1인칭 실제 경험(막힌 지점) 1문단.
- FAQ 5개, 관련검색어 8개, 메타 디스크립션 출력.
- HowTo 스키마용 단계 데이터를 별도 배열로 출력.
- 출력: {제목, 메타, 섹션[], steps[], faq[], 관련검색어[], 태그[], 이미지슬롯[]}
```
**체크:** 5~8단계? 동사 시작? 오류해결 섹션? HowTo 스키마 steps 배열?

### 유형 C. 비교/선택형 (Comparison) ⭐ 고수익

**트리거:** "A vs B", "○○ 차이", "○○ 비교", "○○ 어떤 게 좋아"
**검색의도:** 결정 직전, 뭘 고를지 (구매·전환 임박)
**근거:** "X vs Y"·"best [카테고리]"는 **구매 직전 고전환 트래픽**, 여러 하단퍼널 키워드 동시 상위. **비교 표는 2026 저활용 기회**(경쟁 덜함). 요약(승자·추천)을 **상단**에.

**글 구조:**
1. 제목(H1)
2. 도입부: **한 줄 결론(누구에게 A, 누구에게 B)** + 30초 요약
3. H2 "한눈에 비교" — **비교 표**(항목 × 후보) 최상단
4. H2 "A 상세 — 장단점 / 누구에게"
5. H2 "B 상세 — 장단점 / 누구에게"
6. H2 "상황별 추천" — "가성비면 A / 성능이면 B" 결정 가이드
7. H2 "가격·조건 정리"
8. H2 FAQ
9. 마무리 + 각 후보 바로가기 CTA(제휴/링크)

**형식 무기:** **비교 표**(핵심), 장단점 표, 상황별 추천 박스, 가격 표
**제목 패턴:** `A vs B 비교 (2026) — 뭐가 나을까? 상황별 추천`
**도입부 패턴:** "결론부터: [상황1]이면 A, [상황2]면 B입니다. 아래 표로 핵심만 비교했습니다."
**E-E-A-T:** 둘 다 써본 경험/기준, 실제 사용 맥락
**CTA:** 각 후보 "자세히 보기/구매" 버튼(제휴링크 자리), 비교표 하단 CTA
**스키마:** `Article` + `FAQPage` (+ 필요시 `Product`)
**이미지 슬롯:** 상단 비교 대표 이미지, 각 후보 이미지

**LLM 프롬프트 템플릿:**
```
너는 한국어 비교/선택 가이드 작가다. "{키워드}"에 대해 구매/선택 직전 독자를 위한 비교 글을 써라.
- 비교 대상: {A}, {B}(, {C})  / 대상 독자: {타깃}
- 도입부 첫 문장에 "결론: 누구에겐 A, 누구에겐 B" 명시.
- 최상단에 '한눈에 비교' 표: 행=비교항목(가격/성능/편의/장점/단점 등), 열=후보.
- 각 후보 상세(장단점 표 + "이런 사람에게").
- '상황별 추천' 결정 가이드 박스(가성비/성능/초보 등 조건별).
- 가격·조건 표, FAQ 5, 관련검색어 8, 메타.
- 각 후보에 CTA 버튼 슬롯([바로가기] 라벨+링크자리) 표기.
- 1인칭 사용 경험/판단 기준 1문단.
- 출력: {제목, 메타, 섹션[], 비교표, faq[], cta[], 관련검색어[], 태그[], 이미지슬롯[]}
```
**체크:** 상단 결론? 비교 표 최상단? 상황별 추천? 후보별 CTA?

### 유형 D. 리뷰/후기형 (Review)

**트리거:** "○○ 후기", "○○ 리뷰", "내돈내산", "사용기", "실사용"
**검색의도:** 진짜 써본 사람 얘기 듣고 사고 싶다
**근거:** 2026엔 **"직접 써본 증거(원본 사진·사용 기간)" 없으면 AI가 배제.** 7단계 프레임(사용 전 상태 → 첫 사용 → 30일 후 결론 → 무엇을 대체 → 실제 비용 → 비추천 대상 → 다시 산다면). **결론(추천/비추천) 먼저, 근거 다음, 추천 마지막.**

**글 구조:**
1. 제목(H1)
2. 도입부: **총평/별점 결론 먼저** + 한 줄 추천/비추천
3. H2 "사용 전 기대 / 왜 샀나"
4. H2 "실제 써보니 (첫인상 + N일 사용)" — 원본 사진 슬롯
5. H2 "장점" / H2 "단점" — 솔직하게(단점 명확히)
6. H2 "이런 사람에게 추천 / 이런 사람은 비추천"
7. H2 "가격·가성비 / 대안 비교"
8. H2 FAQ
9. 마무리(다시 산다면?) + 구매 CTA

**형식 무기:** 별점/총평 박스, 장단점 표, 원본 사진 슬롯(필수), 대안 비교 미니표
**제목 패턴:** `○○ 솔직 후기 (N일 사용) — 장단점과 살 만한지`
**도입부 패턴:** "결론부터: 별점 ★★★★☆. [핵심 한 줄 평]. N일 써본 솔직 후기입니다."
**E-E-A-T:** ★핵심 — 사용 기간 명시, 원본 사진/스샷, 구체적 실사용 디테일, 단점 솔직
**CTA:** "최저가 확인/구매" 버튼(제휴), 대안 제품 버튼
**스키마:** `Review` (+ `Article`)
**이미지 슬롯:** 원본 실사용 사진 다수(중요), 상단 대표컷

**LLM 프롬프트 템플릿:**
```
너는 한국어 제품/서비스 리뷰어다. "{키워드}"에 대한 실사용 후기 글을 써라.
- 사용 기간: {기간}, 사용 맥락: {맥락}, 대상 독자: {타깃}
- 도입부 첫 문장에 총평(별점)+한 줄 결론.
- 구조: 왜 샀나 → 실사용(첫인상+N일) → 장점 → 단점(솔직히) → 추천/비추천 대상 → 가격/대안 → FAQ → 다시 산다면.
- 원본 사진 슬롯 여러 개 표기([사진 슬롯: 무엇을 찍을지]).
- 단점은 최소 2개 구체적으로. 과장 금지, 1인칭 경험 디테일 풍부히.
- 별점(5점), 장단점 표, 대안 1~2개 미니비교, FAQ 5, 관련검색어 8, 메타.
- 구매 CTA 버튼 슬롯 표기.
- 출력: {제목, 메타, 별점, 섹션[], 장단점, faq[], cta[], 관련검색어[], 태그[], 이미지슬롯[]}
```
**체크:** 총평 먼저? 사용기간 명시? 원본 사진 슬롯? 단점 솔직? Review 스키마?

---

### 유형 E. 리스트형 (Listicle / Top N) ⭐ AI 인용 최강

**트리거:** "best ○○", "○○ 추천 N", "○○ TOP", "○○ 순위", "○○ 모음", "○가지"
**검색의도:** 큐레이션된 선택지를 한눈에
**근거:** **리스트형 = AI 인용의 21.9%, "best X"는 ChatGPT 인용의 43.8%로 최다.** 모듈형·토큰 효율 구조라 AI가 좋아함. **답 먼저 문단 + 번호 리스트**, 항목 **10~20개**(경쟁 SERP 보고 조정), **entity-rich H3**, `ItemList`+`FAQ` 스키마.

**글 구조:**
1. 제목(H1, "○○ 추천 N선")
2. 도입부: **결론 요약**(1등 or 상황별 베스트) + 선정 기준
3. H2 "선정 기준" (짧게, 신뢰도)
4. H2 "○○ 추천 리스트"
   - 각 항목 = **H3(항목명, 엔티티)** + 한 줄 요약(결론) + 특징/장단점 + 미니표 + CTA
   - 번호 순서, 항목당 100~200자
5. H2 "한눈에 비교 표"(전체 항목 요약)
6. H2 "어떻게 고를까 / 상황별 추천"
7. H2 FAQ
8. 마무리 + 각 항목 CTA

**형식 무기:** 번호+H3 항목, 항목별 미니표, 전체 비교 표, 상황별 추천 박스
**제목 패턴:** `○○ 추천 BEST 7 (2026) — 상황별 순위 총정리`
**도입부 패턴:** "결론부터: 종합 1위는 [X], 가성비는 [Y]입니다. 아래 7개를 기준별로 정리했습니다."
**E-E-A-T:** 선정 기준 명시, 실제 사용/조사 근거, 저자 관점
**CTA:** 각 항목 "자세히/구매" 버튼(제휴), 상단 요약 CTA
**스키마:** `ItemList` + `FAQPage` (+ 리뷰 성격이면 항목별 `Product`)
**이미지 슬롯:** 각 항목 이미지, 상단 대표 콜라주

**LLM 프롬프트 템플릿:**
```
너는 한국어 큐레이션 리스티클 작가다. "{키워드}"에 대한 추천 리스트 글을 써라.
- 항목 수: {N}(기본 7~10), 대상 독자: {타깃}, 정렬 기준: {기준}
- 도입부 첫 문장에 "종합 1위 + 상황별 베스트" 결론 요약.
- '선정 기준' 섹션으로 신뢰도 확보.
- 각 항목: H3(항목명) + 첫 문장 한 줄 결론 + 특징/장단점 + 미니표 + CTA 슬롯. 항목당 100~200자.
- 마지막에 전체 '한눈에 비교' 표 + '상황별 추천' 박스.
- 1인칭 조사/사용 근거 1문단, FAQ 5, 관련검색어 8, 메타.
- ItemList 스키마용 항목 배열 별도 출력.
- 출력: {제목, 메타, 선정기준, items[](각 name/요약/장단점/cta), 비교표, faq[], 관련검색어[], 태그[], 이미지슬롯[]}
```
**체크:** 결론 요약 먼저? 항목이 H3+엔티티? 선정기준? 전체 비교표? ItemList 스키마?

### 유형 F. 이슈/트렌드형 (News / Trending)

**트리거:** 인물·작품·사건, 실시간 검색어, "○○ 논란", "○○ 근황", "○○ 총정리"
**검색의도:** 지금 화제인 걸 빠르게 파악
**근거:** 검색량 급등 타이밍 트래픽. **신선도가 생명**(업데이트 표기), 빠른 요약 + 타임라인 + FAQ 구조가 유효. 단, 사실 정확성·출처 중요(신뢰도).

**글 구조:**
1. 제목(H1)
2. 도입부: **핵심 요약 3줄**(무슨 일인지 먼저) + 업데이트 날짜
3. H2 "무슨 일? (핵심 요약)" — 결론 먼저
4. H2 "배경 / 전체 맥락"
5. H2 "타임라인" (날짜별 정리)
6. H2 "반응 / 쟁점 / 논란"
7. H2 "앞으로 전망"
8. H2 FAQ
9. 마무리(업데이트 예고) + 관련 이슈 CTA

**형식 무기:** 요약 박스, 타임라인, 인용/출처 링크, FAQ
**제목 패턴:** `○○ 총정리 — 무슨 일인지 한눈에 (2026.MM.DD 업데이트)`
**도입부 패턴:** "핵심만: [한 줄 요약]. [날짜] 기준 현재까지 알려진 내용을 정리했습니다."
**E-E-A-T:** 출처 명시(공신력 링크), 사실/추측 구분
**CTA:** 원문·공식 링크 버튼, 관련 이슈 글 버튼
**스키마:** `Article`(NewsArticle 성격) + `FAQPage`
**이미지 슬롯:** 상단 대표 이미지(생성/저작권 안전), 타임라인 보조

**LLM 프롬프트 템플릿:**
```
너는 한국어 이슈 정리 에디터다. "{키워드}"에 대한 이슈/트렌드 정리 글을 써라.
- 오늘 날짜: {날짜} 기준으로 최신성 표기("최종 업데이트: {날짜}").
- 도입부에 핵심 요약 3줄(무슨 일인지 결론 먼저).
- 구조: 핵심요약 → 배경 → 타임라인(날짜별) → 반응/쟁점 → 전망 → FAQ.
- 사실과 추측을 구분하고, 확인 안 된 내용은 "~로 알려짐/추정" 표기. 출처 링크 슬롯 표기.
- 자극적 허위·단정 금지. FAQ 5, 관련검색어 8, 메타.
- 출력: {제목, 메타, 요약3줄, 섹션[], 타임라인[], faq[], 출처[], 관련검색어[], 태그[], 이미지슬롯[]}
```
**체크:** 핵심요약 먼저? 업데이트 날짜? 타임라인? 출처 표기? 사실/추측 구분?
