# 블로그 자동 작성 크롬 확장 프로그램 — 상세 기획서

> 목적: 워드프레스 / 블로거닷컴 블로그에 **애드센스 부업용 SEO 최적화 글**을
> 키워드만 넣으면 자동 생성 → 이미지·썸네일 생성 → 클릭 버튼 삽입 →
> 블로그 편집 페이지에 자동 입력까지 해주는 크롬 확장 프로그램.

작성일: 2026-07-15 · 버전: v0.1 (초안)

---

## 1. 프로젝트 개요

### 1.1 한 줄 정의
블로그 편집 페이지를 열어둔 상태에서, 크롬 확장 사이드패널에 키워드를 넣고
"글 생성"을 누르면 → **SEO 최적화된 본문 + 썸네일 + 본문 이미지 + 클릭 버튼(CTA)**이
편집기에 자동으로 입력되는 도구.

### 1.2 핵심 가치
- **속도**: 키워드 → 완성형 초안까지 1~2분
- **SEO**: 2026 트렌드 기반 노출 문법(AI 오버뷰 인용 최적화, 결론 먼저, 유형별 구조, 스키마, 체류시간↑, 내부/외부 링크 버튼)을 자동 적용 — 옛날식 키워드 도배가 아니라 검색의도에 맞는 구조가 핵심
- **디자인**: 표·박스·CTA 버튼·이미지가 들어간 "이쁜 글"을 자동 구성
- **검수 가능**: 자동 발행이 아니라 편집기에 "입력"까지만 → 사용자가 검수 후 발행 (애드센스 정책 리스크 완화)

### 1.3 대상 플랫폼
- **워드프레스** (Gutenberg 블록 편집기 / 클래식 에디터)
- **블로거닷컴** (Blogger 기본 편집기)
- 확장 UI에서 **탭으로 플랫폼 전환** 가능 (둘 다 지원)

## 2. 핵심 컨셉 & 설계 원칙

1. **생성은 자동, 발행은 수동(기본값)** — 대량 자동발행은 애드센스/구글 검색 스팸 제재 위험이 있으므로, 기본 흐름은 "편집기 자동 입력 → 사용자 검수 → 사용자가 발행". API 직접 발행은 옵션으로 제공.
2. **삽입 방식 이중화**
   - (메인) 편집기 DOM 자동 입력 — 사용자가 원하는 흐름, 검수 가능
   - (옵션) REST API / Blogger API 직접 발행 — 빠른 발행이 필요할 때
3. **플랫폼 독립적 HTML** — CTA 버튼·박스는 인라인 스타일로 생성해 어느 편집기에 붙어도 깨지지 않게.
4. **모듈화된 파이프라인** — 트렌드 → 키워드 확장 → 글 생성 → 이미지 생성 → HTML 조립 → 삽입. 각 단계 독립 교체 가능.

## 3. 사용자 시나리오 (핵심 워크플로우)

```
1. 사용자가 블로그 편집 페이지를 연다 (워드프레스 글쓰기 / 블로거 새 글)
2. 크롬 확장 사이드패널을 연다
3. 상단 탭에서 플랫폼 선택 (워드프레스 / 블로거)
4. 키워드 입력  ── 또는 ──  "트렌드 추천" 클릭 → 추천 키워드 카드 중 선택
5. 옵션 선택: 글 장르(연예이슈/정보성/리뷰/How-to), 글 길이, 톤, 이미지 개수
6. [글 생성] 클릭
     → (진행상황 표시) 키워드 확장 → 본문 생성 → 썸네일 생성 → 본문 이미지 생성
7. 미리보기 확인 (사이드패널 내 프리뷰)
8. [편집기에 입력] 클릭 → 편집기에 본문+이미지+버튼 자동 삽입
9. 사용자가 검수/수정 후 직접 발행   (또는 [API로 바로 발행] 옵션)
```

## 5. 기능 상세

### 5.1 키워드 / 트렌드 엔진

**모드 A — 직접 입력**
- 사용자가 씨앗 키워드 입력 (예: "참교육 김무열")
- LLM이 연관·롱테일 키워드 20~40개 확장
- 검색의도별 클러스터링: 정보형 / 이슈형 / 비교형 / FAQ형
- 각 키워드에 대해 제목안 3개 자동 제안

**모드 B — 트렌드 추천**
- 소스:
  - **네이버 데이터랩 검색어 트렌드 API** (공식·무료, 한국 트렌드 핵심)
  - **구글 트렌드** (비공식, 불안정 → 보조)
  - **구글/네이버 자동완성** 스크래핑 (실시간 롱테일 확보)
- LLM이 "지금 검색량 상승 가능성 + 경쟁 강도"를 추정해 점수화
- 결과: `키워드 + 예상 검색의도 + 추천 제목 3개`를 카드 형태로 제시

> 출력물 예: 카드 리스트 → 사용자가 하나 선택 → 글 생성으로 연결

### 5.2 글 생성 엔진 (SEO 최적화 — 최우선 기능) ★핵심

> ⚠️ 중요 방향: "키워드를 최대한 많이 도배하는 옛날식 글"은 이제 오히려 불리합니다.
> **2026년 트렌드 리서치 결과**를 반영해, 이 엔진은 제가 최신 노출 방식을 학습해
> **주제/유형(검색의도)별로 지금 노출이 잘 되는 작성 공식**을 자동 선택·적용합니다.

#### 5.2.1 2026년 노출 원칙 (리서치 기반 · 프롬프트로 강제)

1. **AI 오버뷰 인용 최우선** — 구글 검색의 ~50% 이상에 AI 요약이 뜨고, "1위 노출"보다 "AI 요약에 인용되는 것"이 트래픽의 새 화폐. AI가 긁어가기 쉬운 형태로 써야 함.
2. **결론 먼저(Answer-first)** — 모든 H2/H3 바로 아래 첫 문장(약 50~60자)에 그 질문의 답/결론을 먼저 제시. AI가 이 첫 문장을 요약에 우선 인용.
3. **섹션 자기완결성** — 각 H2/H3는 하나의 하위 질문에 그 안에서 완결. 답을 여러 섹션에 쪼개지 않음.
4. **의도-형식 매칭** — 정의형=문단, 방법형=번호 리스트, 비교형=표. (아래 5.2.2 참조)
5. **구조화 데이터(스키마) 자동 삽입** — `Article` + 유형별 `FAQPage`/`HowType` JSON-LD. 스키마 있는 글이 AI 오버뷰·피처드 스니펫에 2~4배 더 인용됨.
6. **E-E-A-T / 경험 신호** — "직접 해봤는가"가 순위 핵심. 내돈내산 후기·실수담·직접 관찰 등 1인칭 경험 문단을 유형에 맞게 삽입(이미지 슬롯과 연동).
7. **신선도 신호** — "최종 업데이트: YYYY-MM-DD" 표기, 본문에 최신 연도(2026) 데이터 반영.
8. **결정을 돕는 콘텐츠** — 단순 정보(=AI가 대체)보다 가격/조건/장단점/체크리스트/사례로 "선택을 돕는" 글이 체류시간·클릭·수익에 유리.
9. **기본 위생** — 본문 1,000자+, 핵심 키워드 자연 반복(과밀 금지), 내부/외부 링크, 이미지 alt에 키워드, H태그 시맨틱 구조.
10. **CTA/클릭 버튼** — 섹션 사이 `▶ 바로가기`형 CTA 박스·관련글 버튼(인라인 스타일)으로 클릭 유도.

#### 5.2.2 유형(검색의도)별 작성 공식 — 엔진이 자동 선택

씨앗 키워드를 넣으면 LLM이 **검색의도를 판별해 아래 유형 중 하나(또는 혼합)를 자동 선택**하고, 그 유형의 구조로 글을 생성합니다. (수동 지정도 가능)

| 유형 | 언제 | 핵심 구조 | 형식 무기 | 스키마 |
|------|------|-----------|-----------|--------|
| **정보/설명형** (What/Why) | "○○란", 개념·원리 | 결론 먼저 문단 → 상세 → 요약 표 | 정의 문단 + 요약표 | Article, FAQPage |
| **방법/How-to형** | "○○하는 법", 절차 | 번호 단계(1·2·3) + 각 단계 결론 먼저 | 번호 리스트 + 체크리스트 | HowTo, FAQPage |
| **비교/선택형** ⭐고수익 | "A vs B", "추천", "차이" | 비교표 → 상황별 추천 → 체크리스트 | **비교 표**(2026 저활용 기회) + 추천 박스 | Article, FAQPage |
| **리뷰/후기형** | "○○ 후기/리뷰" | 내돈내산 경험 → 장단점 표 → 총평 | 경험 문단 + 사진 슬롯 + 별점/장단점 표 | Review, Article |
| **리스트형** (TOP N) | "○○ 추천 5", 모음 | 넘버링 항목 + 각 항목 요약 | 넘버링 + 항목별 미니표 | ItemList, Article |
| **이슈/트렌드형** | 연예·시사·실시간 | 빠른 요약 → 핵심 정보 → 타임라인 → FAQ | 요약 박스 + 타임라인 + FAQ | Article, FAQPage |

각 유형별로 **제목 패턴 · 도입부 패턴 · 섹션 순서 · CTA 위치 · 스키마**를 프리셋으로 저장 → 프롬프트 조립 시 자동 적용.

#### 5.2.3 공통 출력 요소
- 제목: 핵심 키워드 앞쪽 + 클릭 유도(숫자·연도·"총정리" 등 유형별)
- 메타 디스크립션 자동 생성
- 하단 FAQ(Q&A) + 연관 검색어
- 태그, alt 텍스트, 내부/외부 링크 자동

**출력 형식:** 구조화된 JSON
```json
{
  "title": "...",
  "metaDescription": "...",
  "sections": [
    {"type": "h2", "text": "..."},
    {"type": "paragraph", "text": "..."},
    {"type": "table", "rows": [...]},
    {"type": "cta_button", "label": "▶ 바로 보기", "url": "..."},
    {"type": "image_slot", "prompt": "이미지 생성용 설명", "alt": "..."}
  ],
  "faq": [{"q": "...", "a": "..."}],
  "relatedKeywords": ["...", "..."],
  "tags": ["...", "..."]
}
```
→ 이 JSON을 HTML로 조립하면 플랫폼 무관하게 삽입 가능.

### 5.3 이미지 · 썸네일 생성

- **썸네일**: 이미지 생성 API로 배경 생성 → 확장 내 Canvas로 제목 텍스트 오버레이(폰트/색/위치 프리셋). 저작권 안전, 클릭률 높은 썸네일 자동 제작.
- **본문 이미지**: `image_slot`마다 섹션 내용에 맞는 이미지 자동 생성/삽입, `alt` 텍스트 자동(이미지 SEO).
- **엔진**: KIE.ai 이미지 모델(Flux 등) 중심.
- 이미지 삽입 방식:
  - 워드프레스: 미디어 라이브러리 업로드(REST) 후 URL 삽입, 또는 base64 → 편집기 삽입
  - 블로거: 이미지 호스팅 후 URL 삽입

### 5.4 편집기 자동 삽입 (Content Script)

**공통:** 생성 JSON → 인라인 스타일 HTML로 변환 → 어댑터가 편집기에 주입.

**워드프레스 어댑터**
- Gutenberg: HTML을 블록으로 파싱해 주입 (또는 "코드 편집기" 모드로 HTML 삽입)
- 클래식 에디터: TinyMCE 본문에 HTML 삽입
- (옵션) WP REST API `/wp-json/wp/v2/posts` + Application Password 로 초안/발행

**블로거 어댑터**
- 편집기를 "HTML 보기" 모드로 전환 → 본문 영역 `innerHTML` 주입
- (옵션) Blogger API v3 + 구글 OAuth 로 초안/발행

**탭 전환:** 사이드패널 상단 탭으로 워드프레스/블로거 선택 → 해당 어댑터·설정 자동 적용.

## 6. API 구성 (사용자 준비 항목)

| 용도 | 서비스 | 비고 |
|------|--------|------|
| 글 생성(LLM) | **KIE.ai** | 키 1개로 LLM 커버 (메인) |
| 이미지·썸네일 | **KIE.ai** | 이미지 모델(Flux 등) |
| 한국 트렌드 | **네이버 데이터랩 API** | 개발자센터 무료 가입, 사실상 필수 |
| 보조 트렌드 | 구글 트렌드 / 자동완성 | 비공식, 불안정 |
| WP 발행(옵션) | WordPress Application Password | API 발행 쓸 때만 |
| 블로거 발행(옵션) | Google OAuth (Blogger API) | API 발행 쓸 때만 |

- 모든 키는 Options 페이지에서 입력 → `chrome.storage.local` 저장.

│ ⚙ 옵션                          │
│  장르 [연예이슈▾] 길이 [보통▾]  │

## 8. 개발 로드맵

| 단계 | 범위 | 산출물 |
|------|------|--------|
| **0. 셋업** | MV3 프로젝트 골격, 사이드패널, Options, 키 저장 | 빈 확장 실행 |
| **1. MVP** | 키워드 직접입력 → KIE로 SEO 글 생성(JSON→HTML) → 워드프레스 편집기 자동입력 | 글부터 동작 |
| **2. 이미지/디자인** | 썸네일 + 본문 이미지 생성/삽입, CTA 버튼·FAQ·표 자동화 | "이쁜 글" 완성 |
| **3. 트렌드** | 네이버 데이터랩 + 자동완성 추천 엔진 | 키워드 추천 |
| **4. 블로거 + 발행** | 블로거 어댑터(탭 전환), WP/Blogger API 발행 옵션 | 양 플랫폼 완성 |
| **5. 고도화** | 장르별 템플릿 확장, 프리셋, 예약/일괄, 품질 리뷰 | 운영 도구화 |

## 9. 리스크 & 대응
